## 1. 청구범위 최대화 4원칙

### 원칙 1 — 기술 중립적 표현
특정 기술명 대신 기능·목적 중심으로 기재하여 기술이 바뀌어도 권리 유지

| 쓰면 안 되는 표현 | 써야 하는 표현 | 이유 |
|---|---|---|
| "GPT-4o를 이용하여" | "언어 생성 모델을 이용하여" | 모델 교체 시 권리 소멸 방지 |
| "네이버 CLOVA OCR로" | "이미지 인식 수단을 이용하여" | 특정 API 종속 방지 |
| "Slack API를 통해" | "협업 도구 서버와 API 연동을 통해" | 도구 변경 시 권리 유지 |
| "결제창에서 실시간으로" | "결제 처리 시점에" | 구현 방식 변경 허용 |

---

### 원칙 2 — 독립항은 최소 요소만
독립항에 조건이 많을수록 권리 범위가 좁아짐. 핵심 단계만 남기고 세부 구현은 종속항으로 내림

```
독립항 (넓음): A → B → C 만 기재
  └ 종속항 1: A → B → C + D (AI 매칭 레이어 추가)
     └ 종속항 2: A → B → C + D + E (월 단위 지자체 정산 추가)
        └ 종속항 3: A → B → C + D + E + F (부서별 예산 분할 추가)
```

독립항이 무효가 되어도 종속항은 살아남음

---

### 원칙 3 — 방법·시스템·기록매체 3세트 출원
동일 발명을 3가지 형태로 청구하여 침해 유형 최대 커버

| 청구 형태 | 보호 대상 | 침해 예시 |
|---|---|---|
| 방법 청구항 | 플랫폼이 수행하는 절차 | 경쟁사가 동일한 처리 흐름을 구현할 때 |
| 시스템 청구항 | 하드웨어·소프트웨어 구성 | 경쟁사가 동일한 서버 구성을 판매할 때 |
| 기록매체 청구항 | 소프트웨어 자체 | 경쟁사가 동일한 프로그램을 배포할 때 |

---

### 원칙 4 — 명세서에 실시예 최대한 많이 기재
청구항은 나중에 보정·추가 가능하지만, 명세서에 없는 내용은 절대 청구 불가
→ 지금 구현 안 한 확장 기능도 아이디어 수준으로 명세서에 기재해둘 것

**지금 넣어야 할 확장 실시예 목록**
- 워케이션 지원금 잔액 실시간 예측 기능
- 복수 지자체 지원금 중복 적용·우선순위 조정
- 임직원 워케이션 선호도 데이터 기반 숙소 자동 추천
- 워케이션 ESG 기여도와 탄소 발자국 연동 산출
- 해외 워케이션 환율 자동 변환 정산
- 워케이션 반복 참여 데이터 기반 번아웃 예측 모델

## 2. 미래 분쟁 리스크 4가지

### 위험 1 — 경쟁사 포위 전략 (Fencing)
경쟁사가 우리 특허 위에 개량발명을 쌓아서 우리가 기능을 업그레이드할 때 막는 전략

```
우리 특허:  지원금 자동매칭 + 선공제
경쟁사 출원: 지원금 자동매칭 + 선공제 + AI 잔액예측  ← 우리가 이 기능 추가하려면 경쟁사 허락 필요
```

**대응책**
- 명세서에 확장 실시예를 미리 기재 (위 목록 참고)
- 서비스 업데이트마다 연속출원으로 새 기능을 우리 특허 포트폴리오에 추가
- 매 6개월 KIPRIS 모니터링으로 경쟁사 출원 조기 감지

---

### 위험 2 — 광범위 청구항 무효
독립항이 너무 넓어서 나중에 발굴된 선행기술에 의해 무효가 됨

**대응책**
- 독립항 → 종속항 다층 구조 설계 (독립항 무효 시 종속항이 방어선)
- 출원 전 깔때기 검색식으로 Level별 선행기술 철저히 확인
- 청구항을 "기술적 특징"이 아닌 "기술적 효과"로도 표현하여 무효 주장에 대한 방어 논리 확보

---

### 위험 3 — 기술 발전으로 청구범위 이탈
기술이 발전해서 우리가 쓴 표현 밖의 방식으로 동일한 효과를 구현하는 제품이 나옴

**대응책**
- 원칙 1(기술 중립적 표현)로 작성하면 상당 부분 방어 가능
- "~수단", "~을 통해", "적어도 하나의" 등 포괄적 표현 사용
- 기술 트렌드 변화 시 연속출원·분할출원으로 새 청구항 추가

---

### 위험 4 — 해외 경쟁사 글로벌 선점
한국 특허는 한국 영토에서만 효력 → 글로벌 확장 시 해외에서 동일 기술로 선점당할 수 있음

**대응책**
- 한국 출원일로부터 **12개월 이내** PCT(국제특허출원) 신청
- PCT 출원 시 한국 출원일을 우선일로 인정 → 전 세계 150개국 동시 출원 효과
- 우선 진출 예정 국가: 일본 (소멸위험 지자체 744개, 워케이션 정책 유사)

## 3. 단계별 IP 로드맵

| 시점 | 액션 | 비용(예상) |
|---|---|---|
| **즉시** | 특허 3건 출원 (변리사 통해) | 600~700만원 |
| **출원 후 3개월** | KIPRIS 특허 모니터링 알림 설정 | 무료 |
| **출원 후 6개월** | 경쟁사 출원 현황 1차 점검 | 자체 수행 |
| **출원 후 12개월** | PCT 국제출원 검토 (일본 우선) | 300~500만원 |
| **서비스 업데이트마다** | 새 기능 연속출원 검토 | 건당 100~200만원 |
| **매 6개월** | 선행기술 모니터링 + 포트폴리오 점검 | 자체 수행 |

## 4. 방어적 공개 대상 목록
특허로 안 잡을 기능은 공개 발표(논문, 블로그, GitHub 등)해서 경쟁사가 특허화하지 못하게 막음

| 기능 | 방어적 공개 권장 이유 |
|---|---|
| 워케이션 숙소 단순 검색·필터링 | 이미 유사 기술 다수 존재, 특허 가치 낮음 |
| 임직원 만족도 설문 수집 | 일반적인 기능, 특허 등록 어려움 |
| 워케이션 일정 캘린더 연동 | 구글 캘린더 등과 유사, 신규성 낮음 |

→ 위 기능들은 기술 블로그나 GitHub에 구현 방식을 공개하면 경쟁사가 해당 기능을 특허로 선점하는 것을 막을 수 있음
